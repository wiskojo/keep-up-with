# Thread Quality Checklist

Read this before sending a medium or deep thread.

## Relevance

- [ ] The thread answers why the user is seeing this now.
- [ ] The event is relevant to the user's stated interests, current subscriptions, or durable memory.
- [ ] The thread adds a real delta, not just a generic summary.
- [ ] Old context is included only when needed to understand the delta.

## Structure

- [ ] Background identifies who is involved and gives enough context for unknown people, startups, repos, papers, or projects.
- [ ] Background says when the event happened, or explains why an older event is surfacing now.
- [ ] What happened / what is it is plain English and short.
- [ ] What's changed / new compares against prior story state, relevant baselines, or comparable events when that comparison is real.
- [ ] Highlights are ordered by importance and stop before low-value details.
- [ ] Sources are collected at the bottom and do not clutter the main channel.

## Evidence

- [ ] The main event source is cited.
- [ ] Comparison, correction, and context sources are cited when used.
- [ ] Each non-obvious factual claim has a source behind it.
- [ ] Observation is separated from interpretation.
- [ ] Unverified claims are labeled as unverified.
- [ ] Limits and uncertainty are stated plainly.

## Usefulness

- [ ] The thread saves the user time compared with reading the raw sources.
- [ ] It avoids internal triage, routing, skipped-item, or process narration.
- [ ] It avoids hype words and unsupported judgment.
- [ ] It uses visuals only when they clarify, verify, or compare.
- [ ] It does not fill space to satisfy a quota.

## Final Pass

- [ ] The first section makes the point clear.
- [ ] The thread is concise enough for the user's likely interest level.
- [ ] The title is factual, not clickbait.
- [ ] Links work and point to the cited sources.
- [ ] If this updates an existing story, append to the existing thread instead of creating a duplicate.
- [ ] After sending, update memory only for durable lessons or preferences.
