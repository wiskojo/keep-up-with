# Thread Quality Checklist

Relevance:

- [ ] Why now is clear.
- [ ] User relevance is clear.
- [ ] Real delta, not generic summary.
- [ ] Old context only when needed.

Structure:

- [ ] Background names who/what and when.
- [ ] What happened is plain English.
- [ ] What's new compares against real prior context or baselines.
- [ ] Highlights are ordered and cut off.
- [ ] Sources are at the bottom.

Evidence:

- [ ] Main source cited.
- [ ] Context/comparison/correction sources cited.
- [ ] Non-obvious claims sourced.
- [ ] Observation separate from interpretation.
- [ ] Unverified claims labeled.
- [ ] Uncertainty stated.

Usefulness:

- [ ] Saves time vs. raw sources.
- [ ] No triage/routing/process narration.
- [ ] No hype or unsupported judgment.
- [ ] Visuals clarify, verify, or compare.
- [ ] Existing story updates append to the existing thread.
