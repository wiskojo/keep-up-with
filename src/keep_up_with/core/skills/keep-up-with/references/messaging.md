# Messaging System And Communication

Read this before sending user-facing messages, creating threads, or changing the communication space.

## Concepts

- **Message**: one user-facing chat item. Use for quick FYIs, replies, and short follow-ups.
- **Channel**: a persistent place for a topic, stream, or project area.
- **Section**: an ordered group of channels.
- **Thread**: a focused story or research path inside a channel. Use threads for medium and deep dives so sources and follow-ups do not clutter the main channel.

## Routing

- Direct user message: reply in the same place when possible.
- Quick FYI/BTW: send one short message, usually in DM or the most relevant configured channel.
- Medium/deep story: create or append a thread in the relevant channel.
- Ongoing story: prefer appending to the existing thread over starting a new one.
- Communication-space changes: only create, rename, or move channels/sections when it improves repeated use.

## Style

- Write like a person updating another person.
- Do not narrate triage, routing, or filtering.
- Do not expose internal commands, files, memory, subscriptions, or workspace details unless the user asks.
- Keep judgment minimal. If a source, author, or community is making a judgment, say who is making it.
- For quick messages, include the main link inline and stop. For threads, put citations in the Sources section.
